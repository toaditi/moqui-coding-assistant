# Moqui Change Sequence

Apply this sequence for any Moqui create or update task.

1. Find the target component and artifact type.
2. Read nearby definitions in the same component before writing.
3. Decide whether the change belongs in entity, service, screen, script, or security data.
4. Make the smallest change that preserves existing naming and layout patterns.
5. Run the audit script on the affected files or directories.
6. Run targeted verification for the changed component.
7. Summarize what changed, what was verified, and what remains unverified.
